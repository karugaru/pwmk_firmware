# プロジェクト全体の方針

## 可能な限り、巨人の肩の上に立つ

このプロジェクトでは、既存のライブラリやフレームワークを活用することを推奨します。
ただし、本プロジェクトのライセンスがMITであるため、適合しない場合は自前で実装することも検討します。
例えば、Cirque社のトラックパッド用のライブラリは、ライセンスが適合しなかったため、自前で実装しました。

## 可能な限り、持続可能性を考慮する

このプロジェクトは、長期的に維持されることを目指しています。
そのため、コードの品質やドキュメントの充実を重視し、将来的に他の人がプロジェクトを引き継ぎやすいようにすることを目指します。
また、依存するライブラリやフレームワークの選定においても、持続可能性を考慮します。
例えば、このプロジェクトはPico-SDKありきで開発されていますが、それはRaspberry Pi 財団やRaspberry Pi Ltdが
Pico-SDKを長期的にサポートすることを期待しているからです。

## AIを信用しすぎない

AIは非常に便利なツールですが、完全ではありません。
特に、コードの生成やドキュメントの作成においては、AIが生成した内容を鵜呑みにせず、必ず人間の目で確認することが重要です。
そのため、このプロジェクトでは、AIを補助的なツールとして活用しつつも、最終的な判断は人間が行うことを重視します。
人間が読んで、確認していないコードはこのプロジェクトには一切含めないようにします。

## コードを読みやすく

ソースコードを読むうえで、前提知識が要求されることが多いです。
特に、ファームウェア特有の処理や、ライブラリ独自の機能を使用している場合は、コードを理解するために多くの時間が必要になります。
そのため、このプロジェクトではコメントを読むだけでコードの内容が理解できるように、コードを読みやすくすることを重視しています。

## 関数名からどのファイルで定義されているのかがわかるように

基本的に、関数名の接頭字として、ファイル名を使用します。例として、`state.c`にある関数は、`state_`から始まるようにします。
どの関数がどのファイルで定義されているのかがわかるようにすることで、コードを理解しやすくなります。

## ドキュメントは日本語 (今は)

このプロジェクトのドキュメントは日本語で記述します。
将来的に英語に移行する可能性もありますが、現時点では日本語で記述することを優先します。

## ライセンス

このプロジェクトはMITライセンスで公開します。
ただし、一部のコードについてはライセンスが適合しない可能性があります。
その場合はファイルの先頭に適切なライセンステキストを必ず記載します。
