# 内部処理ドキュメント

## 処理フロー

### 前提

- 基本的に、デバイスへの入力はタイマー割り込みで処理されます。
- BLE通信とペリフェラル処理はメインループで処理されます。
- タイマー割り込み処理はメインループでの処理を要求することができ、メインループは要求がある場合にのみ対応します。
- この構造はbtstackを使用する上でのベストプラクティスに従っています。

### データレベル

- キーマトリクスのスキャン結果は、GPIOの状態を表す配列として保存されます。
- スキャン結果から、キーコードとその状態（押下/解放）を抽出し、内部HID状態として保存・管理します。
- 内部HID状態は、現在のキー状態やマウスの移動量、ホイールの移動量などを含むデータ構造です。
- HIDレポートは、内部HID状態に基づいて生成されるバイト列です。

- ユーザーコードで独自のイベント処理を行う場合、内部HID状態を更新し、イベント発生フラグを立てる必要があります。

### 処理フロー概略

1. タイマー割り込み：キーマトリクススキャン → キー状態変更検出 → 内部HID状態更新 → メインループ処理要求
2. メインループ：ペリフェラル処理 → 内部HID状態更新 → HIDレポート送信要求
3. HIDレポート送信：内部HID状態からレポート生成 → BLE送信

## 処理フロー詳細

1. タイマー割り込み処理
   - 一定周期で呼び出され、以下の処理を行います。
   - キーマトリクスのスキャンを行い、キー入力検出を行います。
     - キー状態に変更があった場合、ユーザ定義イベント処理と標準イベント処理を実行します。
     - これは内部HID状態を更新する可能性があります。
   - 内部HID状態に変更があれば、メインループでのイベント処理を行うよう要求します。
   - ペリフェラル処理が必要な場合、メインループでのイベント処理を行うよう要求します。
2. メインループ処理
   - ペリフェラル処理を行います。
   - ユーザ定義メインループ用イベント処理を実行します。(TODO:未実装)
   - これらは内部HID状態を更新する可能性があります。
   - HIDレポート送信の必要があれば、HIDレポートを送信要求を行います。
3. HIDレポート送信処理
   - btstackのイベントハンドラ処理で呼び出されます。
   - 最新のHID状態に基づいてHIDレポートを生成し、送信します。
