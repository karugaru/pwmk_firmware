
# プロファイルシステム

ユーザ設定はプロファイル単位で管理します。プロファイルを切り替えることで、ボード設定、キーマップ、ユーザコード、CMake のビルド設定をまとめて切り替えられます。

## 概要

- 各プロファイルは `users/<profile>` に配置します。
- プロファイルごとの設定は `profile.yaml` に記述します。
- ユーザコードは `users/<profile>/users.c` に記述します。
- 現在選択中のプロファイル名は `users/.current_profile` に保存されます。
- ビルド時には、選択中プロファイルの内容から CMake 設定と C ソースが自動生成されます。

現行の標準プロファイルは `users/remopicon_v1` です。

## ディレクトリ構成

プロファイルは以下の構成を前提とします。

```text
users/
    .current_profile
    <profile_name>/
        profile.yaml
        users.c
    <another_profile_name>/
        profile.yaml
        users.c
    ...
```

`profile.yaml` には以下のような内容を定義します。

- `cmake`: `PICO_BOARD`、USB/BLE 有効化などのビルド設定
- `board`: 行列ピン、LED ピン、レイアウトなどのボード設定
- `keymap`: キーマップとユーザ定義キーコード
- `settings`: スリープ、LED、Pinnacle、BLE などの動作設定

## プロファイルを選択する

ビルドに使用するプロファイルを選択します。

```powershell
uv run tools/pwmk.py profile <profile_name>
```

このコマンドを実行すると、以下が行われます。

- `users/.current_profile` に選択したプロファイル名が保存されます。
- `build/` ディレクトリが削除され、次回ビルド時に再構成されます。

現在のプロファイル選択を解除する場合は、以下を実行します。

```powershell
uv run tools/pwmk.py profile --clear
```

## ビルド時の動作

`pwmk build` または VS Code の CMake ビルドでは、選択中プロファイルをもとに自動生成が行われます。

処理の流れは以下の通りです。

1. CMake の先頭で `pwmk generate` が実行される
2. `users/.current_profile` から現在のプロファイル名を読み込む
3. `users/<profile>/profile.yaml` を読み込む
4. ビルドディレクトリ配下に CMake 設定と C ソースを生成する
5. 生成結果を使って通常のファームウェアビルドを続行する

生成される主なファイルは以下です。

- `build/.../generated/profile/profile.cmake`
- `build/.../generated/profile/src/settings/*.c`
- `build/.../generated/profile/src/settings/*.h`

## エラー時の挙動

- プロファイル未選択の状態で `pwmk build` を実行するとエラーになります。
- `profile.yaml` の構造や値が不正な場合は、生成時にエラーになります。
- 指定したプロファイルに `profile.yaml` または `users.c` が存在しない場合はエラーになります。

ビルドに失敗した場合は、まず以下を確認してください。

- `users/.current_profile` に正しいプロファイルが保存されているか
- `users/<profile>/profile.yaml` が存在し、内容が正しいか
- `users/<profile>/users.c` が存在するか
