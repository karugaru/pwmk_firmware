# ビルドテスト手順

## はじめに

このガイドでは、Docker を使って CLI ビルドを検証する手順を説明します。
ビルド本体は [tools/build.py](../tools/build.py) が行い、この手順ではその実行をコンテナ内で自動化します。

## 1. 前提環境

この手順は次の前提で動作します。

- Linux 環境に入っていること
- Dockerと`python3` が使えること

## 2. ビルドテストの実行

リポジトリルートで次のコマンドを実行します。

```bash
python3 tools/test_build.py
```

実行を開始すると確認なしでビルドが進行するのでご注意ください。

このスクリプトは次の処理を行います。

- 公式の `ubuntu:24.04` コンテナを起動する
- ソースツリーを `/workspace` にマウントする
- コンテナ内で `python3` を導入する
- コンテナ内でビルドを実行する

## 3. 期待する結果

ビルドが成功すると、コンテナ内のログ末尾で `pwmk.elf` のリンク完了とメモリ使用量が表示されます。
